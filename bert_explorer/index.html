Under the Hood: How BERT Embeddings Work

1. From Static to Contextual Vectors

Traditional embeddings like Word2Vec or GloVe assign a single mathematical vector to each word. This works fine for "computer," but completely fails for polysemous words like "bank".

BERT uses a Transformer encoder. Instead of looking up a static vector, BERT processes the *entire sentence simultaneously* in both directions (left-to-right and right-to-left). As a result, the embedding for "bank" dynamically morphs, adjusting its dimensions depending on the surrounding tokens!

2. Subword Tokenization (WordPiece)

BERT doesn't process whole words directly. It breaks text down into subword units called WordPieces. This prevents "Out of Vocabulary" errors.

Common words stay intact, while rare or complex words are split into pieces marked with a prefix like ##. BERT also prefixes sentences with a special classification token [CLS] and divides segments using [SEP].

3. Bidirectional Attention (Self-Attention)

How does the context get injected? Through Self-Attention. Every token in the input sentence determines how much it should "pay attention" to every other token.

For instance, in the sentence: "The animal didn't cross the street because it was tired", the attention mechanism connects the token "it" heavily to "animal". If we change it to "because it was too wide", attention shifts "it" to connect with "street". This attention-driven blending generates the final contextual embedding vector.

4. Measuring Meaning: Cosine Similarity

Embeddings are arrays of numbers (vectors) in a high-dimensional space (e.g., 384 dimensions for our local model). We compare their meaning using Cosine Similarity, which measures the angle between these vectors.

If two vectors point in almost the same direction, their similarity is close to 1.0 (or 100%). If they are orthogonal (unrelated), similarity is near 0.0.
